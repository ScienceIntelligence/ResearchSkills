## Situation
AlphaEarth hackathon with 3 people (Rushi coding, Xihan labels, KW web). Xihan's label extraction tool failed; most other teams likely to produce static summary+picture outputs; limited time (Fri-Sun).
## Action
Split the team: Rushi focuses on AlphaEarth Earth Engine extraction + kNN/linear probe classifier for Texas wildfire + drought change detection; Xihan pivots from the failed label tool to alternative datasets (NOAA fire perimeters, ERA5-Land drought indices); KW builds an interactive web app on existing Google Cloud server + domain with map-based coordinate input, letting judges click any global point and see time-series outputs. Issue a QR code at final presentation for live mobile access.
## Outcome
Interactive web-based demo differentiated the team from static-output competitors; QR-code live demo on judges' phones was a structural advantage regardless of whether the scientific novelty exceeded baselines.
## Lesson
When scientific novelty is uncertain and compute/data are limited, a modest scientific result wrapped in an interactive, mobile-accessible interface can out-present a stronger result shown as static figures. The scientific bar for hackathons is lower than the presentation bar.
## Retrieval Cues
- Hackathon with foundation models + limited labels
- Team with ≥1 web developer and a pre-existing cloud server
- Need to stand out among static-poster presentations
