## Fact

The Evaluation pillar is intended to be a benchmark plus environment for AI scientist agents, not merely a static evaluation suite.

The intended environment is pluralistic: simulated laboratories, real literature, static tasks, dynamic scenarios, real tools, and simulators may all be included. The important design constraint is that evaluation should expose agents to research-world conditions, not only fixed benchmark questions.

## Evidence

The scientist rejected prior AI-generated specificity and clarified that this pillar should support AI agent scientists with both benchmark and environment. When asked what the research world contains, they identified simulated labs and real literature, then clarified that dynamic/static and simulated/real forms should all be possible.

## Expiry Signal

Expires if the project narrows Evaluation to a fixed benchmark format, such as static QA, game-resistant tests, literature-only tasks, or a single environment type rather than pluralistic research worlds.
